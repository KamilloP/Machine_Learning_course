Homework 1¶

Task A¶

Content¶

Calculate Shapley values for player A given the following value function

v() = 0
v(A) = 20
v(B) = 20
v(C) = 60
v(A,B) = 60
v(A,C) = 70
v(B,C) = 70
v(A,B,C) = 100

Solution¶

SHAP formula for fair reward sharing is $$ \phi_j = \frac{1}{|P|!} \sum_{\pi \in \Pi} V(S_{j}^{\pi} \cup j) - V(S_{j}^{\pi}) $$ or equivalently $$ \phi_j = \sum_{S \subset P \setminus \{j\}} \frac{|S|! (|P| - |S| - 1)!}{|P|!} V(S \cup j) - V(S) $$ Of course we want to find $\phi_A$ in our case. Less calculation is required in second approach, so let's use it.

$$ \phi_A = \frac{1}{3!} \sum_{S \subset P \setminus \{j\}} |S|! (|P| - |S| - 1)! (V(S \cup j) - V(S)) = $$ $$ |\textit{our next subsets $S$ are:} \emptyset, \{B\}, \{C\}, \{B,C\}| = $$ $$ \frac{1}{6} (2 (20-0) + 1(60-20) + 1(70-60) + 2(100-70)) = \frac{40+40+10+60}{6} = \frac{150}{6} = 25 $$

Part 1¶

I selected the CatBoostClassifier model because it achieved satisfactory performance and the dataset is imbalanced. I computed Shapley values for the model; however, the results differ significantly from those obtained using the SHAP package. The magnitude of these differences suggests that they are not merely due to approximation error.

Important design choice concerns the definition of the model output used in the Shapley value computation. In my case, the model output is the predicted probability of the positive class. An alternative would be to use the predicted class label (i.e., the class with maximum probability). However, Shapley values are theoretically defined for real-valued model outputs, which makes probabilities a more appropriate choice than hard class labels.

An additional issue arises from label encoding. I subtracted 1 from the original labels to obtain values in {0,1} instead of {1,2}, as used in the dataset. While this transformation does not change the classification boundary, it affects the numerical scale of the model output and therefore influences the absolute values of the computed Shapley attributions. This must be taken into account when comparing results with those obtained from the SHAP package.

# 4) Compute Shapley values (approximation)
def f(X_):
    return cb.predict_proba(X_)[:,1]

def f_s(f, X_, obs_, S):
    X2 = X_.copy()
    if len(S) > 0:
        X2[:, S] = obs_[S][None, :]  # broadcast
    return f(X2).mean()

def subsetFromPerm(perm, j):
    idx = np.where(perm == j)[0][0]
    return perm[:idx]

def Shapley(f_, X_, obs_, nrPermutations=20):
    n = X_.shape[1]
    res = np.zeros(n)
    for j in range(n):
        sum_ = 0
        for _ in range(nrPermutations):
            perm = np.random.permutation(n)
            S = subsetFromPerm(perm, j)
            sum_ += f_s(f_, X_, obs_, np.append(S, j)) - f_s(f_, X_, obs_, S)
        sum_ /= nrPermutations
        res[j] = sum_
    return res
